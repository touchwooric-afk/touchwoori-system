# 고등부 영수증 관리 시스템 -- Phase 분리 계획

> Phase별로 나눠서 각각 "진짜 동작하는 제품"을 만듭니다.

---

## Phase 1: MVP (예상 5~6주)

### 목표
교사 제출 → 회계 교사 승인 → 장부 자동 반영 → 감사용 PDF 출력.
마스터가 사용자 권한과 시스템 전체를 관리할 수 있는 완전한 워크플로.

### 기능

**공통**
- [ ] 계정 신청 (아이디/이메일 + 비밀번호 + 실명 + 직분) → 승인 대기
- [ ] 비밀번호 재설정 (이메일 기반, Supabase Auth)
- [ ] 역할별 대시보드 (홈 화면)
- [ ] 내 정보 페이지: 이름, 이메일, 직분, 부서명, 비밀번호 변경, 내 제출 목록 바로가기
- [ ] 영수증 이미지 확대 뷰 (클릭/탭 확대, 모바일 핀치 줌)
- [ ] 영수증 이미지 관리: 개별 다운로드, 교체(새 사진 대체), 삭제

**마스터 (master)** — 시스템 관리자 + 담당 목사
- [ ] 신청 대기자 목록 → 승인 + 시스템 역할(teacher/accountant) 부여
- [ ] 사용자 비활성화 (접근 차단)
- [ ] 카테고리(분류) 관리: 추가/수정/비활성화 (수입·지출 구분)
- [ ] 직분 관리: 기본 목록 제공 + 추가/수정/비활성화
- [ ] 결산기 생성: 제목, 시작일, 종료일 직접 설정
- [ ] 모든 부서의 모든 데이터 CRUD (최고 권한)
- [ ] 회계 교사의 모든 기능 포함

**회계 교사 (accountant)** — 자기 부서 장부 관리
- [ ] 미승인(pending) 영수증 목록 + 이미지 확대 검토
- [ ] OCR 결과 검토·수정 후 승인 / 반려
- [ ] 승인 시 장부항목 자동 생성 (source: "receipt")
- [ ] 특수 장부 생성·종료 (수련회, 전도축제 등)
- [ ] 장부항목 직접 추가·수정·삭제 (엑셀처럼 행 단위 편집)
- [ ] 영수증 직접 입력 (폴더 일괄 업로드 + OCR 포함)
- [ ] 장부 조회 (장부 전환, 기간·분류 필터) + 행별 누적 잔액 표시
- [ ] **기간별 결산 조회 화면**: 시작일~종료일 지정 → 카테고리별 수입·지출 합산 표시
- [ ] 감사용 결산 PDF: 기간 직접 지정, 날짜순 정렬, 합계표+영수증 최소 페이지
- [ ] 과거 결산 이력 조회 + PDF 재출력
- [ ] 엑셀 가져오기/내보내기

**교사 (teacher)** — 자기 부서 열람 + 영수증 제출
- [ ] 시스템 URL 접속 → 계정 신청 → 승인 대기
- [ ] 승인 후 로그인
- [ ] 영수증 사진 업로드 + OCR 자동 인식 (날짜·금액)
- [ ] 내용·분류 확인·수정 후 제출
- [ ] 자신의 제출 내역 조회 (상태: 대기중/승인됨/반려됨)
- [ ] 장부 열람 (읽기 전용, 편집 불가)
- [ ] 결산 PDF 열람·다운로드·인쇄 (자기 부서)

> **직분(담당교역자, 부장교사, 총무교사 등)은 표시용일 뿐, 시스템 역할이 teacher면 열람+제출만 가능.**

### 사용 데이터
- User, Ledger, LedgerEntry, Settlement, Receipt (status 포함), Category, Position, ExcelSync

### 인증
- Supabase Auth (무료, 50,000 MAU까지)
- 계정 신청 (이메일 + 실명 + 비밀번호) → master 승인 후 사용 가능
- 역할(role)은 master가 승인 시 부여
- pending 상태는 "승인 대기중" 화면만 표시

### "진짜 제품" 체크리스트
- [ ] Supabase Auth 실제 연동 (하드코딩 비밀번호 X)
- [ ] 역할별 API 접근 제어 구현
- [ ] 실제 Supabase DB + Storage 연결
- [ ] 실제 서버 배포 (localhost X)
- [ ] 스마트폰 브라우저에서 제출 UI 정상 동작

```
이 PRD를 읽고 Phase 1을 구현해주세요.

@PRD/04_PROJECT_SPEC.md

Phase 1 범위:

[인증·권한]
- Supabase Auth 연동 (아이디/이메일 + 비밀번호 + 실명 + 직분)
- 계정 신청 → pending 상태 → master 승인 + 시스템 역할 부여 → active
- 비밀번호 재설정 (이메일 기반, Supabase Auth)
- 역할(role) 기반 접근 제어: master / accountant / teacher
- pending 상태는 "승인 대기중" 화면만 표시 (기능 접근 불가)
- 소속 부서 선택 UI는 Phase 1에서 미구현 ("고등부" 자동 배정)

[공통 기능]
- 역할별 대시보드 (master: 대기자 수·전체 현황, accountant: 미승인 건수·잔액, teacher: 빠른 제출)
- 내 정보 페이지 (이름, 이메일, 직분, 부서명, 비밀번호 변경, 내 제출 목록)
- 영수증 이미지 확대 뷰 (클릭/탭 확대, 모바일 핀치 줌)
- 영수증 이미지 관리: 개별 다운로드, 교체(새 사진 대체), 삭제

[master 전용]
- 신청 대기자 승인 + 시스템 역할 부여 (teacher/accountant)
- 사용자 비활성화 (접근 차단)
- 카테고리(분류) 관리: 추가/수정/비활성화 (수입·지출 구분)
- 직분 관리: 기본 목록 제공 + 추가/수정/비활성화
- 결산기 생성 (제목, 시작일, 종료일 자유 설정)
- 모든 부서의 모든 데이터 CRUD (최고 권한)

[회계 교사 기능]
- 미승인 영수증 목록 + 이미지 확대 검토 + 승인/반려
- 승인 시 장부항목 자동 생성 (approved 상태만, source: "receipt")
- 장부 관리: 전체회기 장부(기본) + 특수 장부(수련회, 전도축제 등) 생성·종료
- 장부항목 직접 추가·수정·삭제 (엑셀처럼 행 단위 편집, source: "manual")
- 장부 조회 (장부 전환, 기간·분류 필터) + 행별 누적 잔액 표시
- 폴더 일괄 업로드 + OCR (Tesseract.js)
- 결산은 LedgerEntry 기준 집계 (Receipt 기준 아님)
- 기간별 결산 조회: 시작일~종료일 지정 → 카테고리별 수입·지출 합산 화면 표시
- 감사용 결산 PDF: 지출만 출력, 기간 직접 지정 + 날짜순 정렬 (04_PROJECT_SPEC.md 레이아웃 규칙 엄수)
- 영수증 없는 장부항목은 PDF에서 "[영수증 없음]" 표시
- 과거 결산 이력 조회 + PDF 재출력
- 엑셀 가져오기/내보내기 (UTF-8 BOM, 컬럼 매핑 미리보기 필수)

[교사 기능 - 스마트폰 최적화]
- 영수증 사진 업로드 + OCR 자동 인식 (자동 저장 금지)
- 내용·분류 확인·수정 후 제출
- 자신의 제출 내역 조회 (상태 확인)
- 장부 열람 (읽기 전용, 편집 불가)

[초기 설정]
- 최초 master 계정: Supabase 대시보드에서 수동 생성 후 DB에 직접 삽입
- 시드 데이터: 카테고리 9종 + 직분 5종 + 전체회기 장부 1개 (supabase/seed.sql)
- Storage 경로: receipts/{department}/{user}/{receipt}_{timestamp}.jpg

[편의 기능]
- 미저장 변경 감지 (모달 닫기 전 확인 다이얼로그)
- 영수증 제출 임시 저장 (localStorage)
- 일괄 승인 (체크박스 선택 → 선택 승인)
- 날짜 입력: 달력 피커 + "오늘"/"어제" 바로가기
- PDF 파일명: {부서명}_{결산기제목}.pdf
- 장부항목 ↔ 영수증 바로가기 (📎 아이콘)
- 영수증 중복 경고 (같은 날짜+금액)

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- pending 사용자는 어떤 API도 호출 불가
- 역할별 API 접근 제어 (teacher가 남의 데이터 접근 불가, 장부는 읽기 전용)
- master만 영수증 삭제·결산기 생성·계정 신청 승인·카테고리·직분 관리 가능
- accountant는 자기 부서 장부 CRUD + 영수증 승인만 가능
- teacher는 자기 부서 열람 + 영수증 제출만 가능 (직분과 무관)
- 영수증 수정/삭제 시 이미 생성된 장부항목은 영향 없음 (독립 관리)
- 결산·PDF는 LedgerEntry 기준, 지출만 출력
- approved 상태 영수증만 결산·PDF에 포함
- PDF는 사용자가 지정한 기간 내 장부항목(LedgerEntry)을 날짜 오름차순 정렬
```

## Phase 2: 편의 기능 (예상 2~3주)

### 전제 조건
- Phase 1이 실제 결산에 사용되고 안정적으로 동작 중

### 기능
- [ ] **반려 후 재제출**: 교사가 반려된 영수증을 수정하여 다시 제출 (rejected → pending)
- [ ] OCR 인식률 개선 — 이미지 전처리 (회전 보정, 밝기·대비 조정)
- [ ] 제출/승인 시 알림 (브라우저 푸시 또는 이메일)
- [ ] 분류별 월간 지출 차트 (막대/도넛)
- [ ] 연도별 결산 비교 요약
- [ ] **PDF 이미지 배치 편집**: 미리보기에서 영수증 이미지 확대·축소·이동 조정 → 저장 → PDF 반영 (Receipt.pdf_crop)
- [ ] PDF 항목 순서 드래그로 재배치
- [ ] 계정 신청 시 소속 부서 선택 UI (Phase 3 다부서 확장 준비)

## Phase 3: 고도화 (예상 미정)

### 전제 조건
- Phase 1 + 2가 6개월 이상 안정적으로 운영 중

### 기능

**감사(auditor) / 교육목사(sub_master) 역할 — ✅ Phase 1 직후 앞당겨 완료**
- [x] **auditor 역할 신설**: 교육위원장 전용, 전체 부서 열람 전용 (수정·삭제·승인 불가)
- [x] **sub_master 역할 신설**: 교육목사 전용, 회원관리+역할부여만 (재정 개입 불가)
- [x] master끼리 상호 삭제 방지 (sub_master는 master 계정 수정 불가)
- [x] 감사 내역 아카이브: 결산기별 감사 문서 첨부 + 메모 기능

> master 역할은 Phase 1에서 이미 존재. Phase 3에서는 다부서 확장에 따라 master 대시보드에 전체 부서 현황 조회 기능 추가.

**부서 관리 (CRUD)** — master 전용
- [ ] 부서 추가/수정/삭제 — `department_id` 하드코딩 → Department 테이블 전환
  - [ ] 부서 추가: 부서명, 소속 회계교사 지정
  - [ ] 부서 삭제 시 해당 부서의 모든 데이터(영수증, 장부, 결산, 사용자 등) 자동 백업 후 비활성화
  - [ ] 백업 데이터 조회·복원 기능

**부서별 데이터 격리**
- [ ] accountant/teacher는 자기 소속 부서 데이터만 조회 가능
- [ ] 부서별 독립 결산 관리
- [ ] 예산 설정 및 초과 경고
- [ ] 결산 보고서 읽기 전용 공유 링크
