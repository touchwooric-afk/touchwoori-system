# 고등부 영수증 관리 시스템 -- 프로젝트 스펙

> AI가 코드를 짤 때 지켜야 할 규칙.
> 이 문서를 AI에게 항상 함께 공유하세요.

## 기술 스택

> **비용 원칙**: 모든 구성요소 무료 플랜 기준. 유료 구독 없음.

| 영역 | 선택 | 비용 | 선택 근거 |
|------|------|------|----------|
| 프레임워크 | Next.js 15 (App Router) | 무료 | 반응형 웹 + API 라우트를 한 프로젝트에서 처리 |
| DB | Supabase (PostgreSQL) | 무료 (500MB) | 파일 저장소 내장, 소규모 운영 충분 |
| 파일 저장 | Supabase Storage | 무료 (1GB) | 영수증 사진 보관, URL 직접 접근 |
| 배포 | Vercel | 무료 | Next.js 공식 배포처, 개인 프로젝트 무료 |
| 스타일링 | Tailwind CSS v4 | 무료 | 반응형 UI 빠르게 구현 |
| 엑셀 처리 | xlsx (SheetJS) | 무료 오픈소스 | 한글 지원, 가져오기/내보내기 처리 |
| PDF 출력 | @react-pdf/renderer | 무료 오픈소스 | 이미지 임베드 가능, 서버에서 생성 |
| OCR | Tesseract.js | 무료 오픈소스 | 브라우저 직접 실행, API 비용 없음. 숫자·금액 인식 양호 |

### 무료 플랜 한도

| 서비스 | 무료 한도 | 예상 사용량 |
|--------|----------|------------|
| Supabase DB | 500MB | 수백 건 영수증 → 수 MB 수준, 충분 |
| Supabase Storage | 1GB | 사진 200장 × 5MB = 1GB, 업로드 시 압축 권장 |
| Vercel | 월 100GB 대역폭 | 소규모 내부 사용, 충분 |

## PDF 레이아웃 규칙 (감사용 결산 PDF)

> AI가 PDF를 구현할 때 반드시 이 규칙을 따라야 합니다.

**권한**: 교사 이상 (자기 부서) — teacher도 PDF 미리보기·다운로드·인쇄 가능. 장부 데이터 수정 권한과 무관.

**핵심 원칙: 종이 최소화**
- 합계표를 별도 페이지로 만들지 않음
- **1페이지 상단**: 합계표 (컴팩트하게)
- **1페이지 하단부터**: 영수증 이미지 2×2 그리드
- 4항목 이하 → A4 1장으로 완료
- 5항목 이상 → 2페이지부터는 영수증만 4개씩

```
[1페이지 구성]
┌────────────────────────────────────────┐
│ 2026년 전반기 결산 | 고등부              │ ← 제목
│ 식비 ₩125,000 / 교통비 ₩45,000 / ...  │ ← 합계표 (한 줄 또는 2줄로 컴팩트)
│ 총계 ₩348,000                          │
├───────────────────┬────────────────────┤
│ 3월 셀 간식비      │ 수련회 교통비       │ ← 항목명 소제목 (10pt)
│  [영수증 이미지]   │  [영수증 이미지]   │ ← 이미지 최대한 크게
│ 2026.03.15        │ 2026.03.20         │
│ ₩48,000           │ ₩12,000            │ ← 최종금액 (굵게)
├───────────────────┼────────────────────┤
│ 교재 구입          │ 사무용품            │
│  [영수증 이미지]   │  [사진 없음]       │ ← 사진 없을 때 텍스트
│ 2026.04.01        │ 2026.04.15         │
│ ₩33,000           │ ₩7,000             │
└───────────────────┴────────────────────┘
```

**규칙:**
- **기간 지정**: 결산기 단위 또는 사용자가 시작일~종료일 직접 지정 가능
- **데이터 기준**: 장부항목(LedgerEntry) 기준으로 집계 (Receipt 기준 아님)
- **지출만 출력**: 수입 항목은 PDF에 포함하지 않음
- **정렬**: 선택한 기간 내 장부항목을 날짜 오름차순 정렬
- **영수증 없는 항목**: 이미지 자리에 "[영수증 없음]" 텍스트 표시
- 이미지: 영역 내 비율 유지, 최대 크기 표시
- 금액: LedgerEntry의 `expense` 표시
- 합계표: 지출 합계만 표시 (카테고리별 지출 합산)
- 기본 용지: A4 세로 (portrait)
- 마지막 페이지 항목 수 < 4이면 빈 칸으로 채워 레이아웃 유지
- **지출 0건**: 해당 기간에 지출 항목이 없으면 PDF를 생성하지 않고 "해당 기간에 지출 항목이 없습니다" 안내 메시지 표시

### PDF 이미지 배치 편집 (Phase 2)

> 자동 배치된 영수증 이미지의 가시성이 떨어질 때, 회계 교사 이상이 이미지별 배치를 조정할 수 있다.

**미리보기 UI (React 컴포넌트, PDF 아님)**
- PDF 생성 전 커스텀 미리보기 화면에서 각 영수증 이미지 셀을 클릭하여 편집 모드 진입
- **확대/축소**: 슬라이더 또는 +/- 버튼 (CSS `transform: scale()`)
- **이동**: 마우스/터치 드래그로 셀 내 표시 영역 조정 (CSS `translate`)
- **초기화**: 원래 자동 배치로 복귀 버튼
- 영수증 없는 항목("[영수증 없음]")은 편집 불가

**저장 구조** — Receipt 테이블의 `pdf_crop` (jsonb, nullable)
```json
{ "scale": 1.5, "offsetX": 20, "offsetY": 40 }
```
- `null`이면 자동 배치 (Phase 1 기본 동작 그대로)
- 값이 있으면 PDF 생성 시 해당 조정값으로 이미지 렌더링
- [배치 저장] 버튼으로 DB에 저장 → 다음 PDF 생성 시에도 유지

**권한**: 교사 이상 (자기 부서). 장부 데이터 읽기 전용인 teacher도 PDF 배치 편집·저장 가능 (장부 데이터 자체는 수정 불가)

## 절대 하지 마 (DO NOT)

- [ ] API 키, Supabase URL을 코드에 직접 쓰지 마 → `.env.local` 사용
- [ ] 기존 DB 스키마를 사전 협의 없이 변경하지 마
- [ ] OCR 결과를 사용자 확인 없이 자동 저장하지 마
- [ ] 미승인(pending) 항목을 결산·PDF에 포함하지 마 → approved만 포함
- [ ] teacher 역할이 다른 사람의 제출 내역을 조회할 수 있게 하지 마
- [ ] pending 상태 사용자가 어떤 기능이든 접근할 수 있게 하지 마 → "승인 대기중" 화면만
- [ ] teacher/accountant가 사용자 관리·결산기 생성을 할 수 있게 하지 마 → master만
- [ ] 계정 신청 화면에서 시스템 역할(role)을 선택할 수 있게 하지 마 → 직분(position)만 선택, role은 master가 승인 시 부여
- [ ] accountant가 영수증을 삭제할 수 있게 하지 마 → master만 삭제 가능
- [ ] teacher가 장부를 수정·삭제할 수 있게 하지 마 → 읽기 전용만 허용
- [ ] 직분(position)에 따라 시스템 권한을 자동 부여하지 마 → 직분과 역할(role)은 별개
- [ ] 엑셀 가져오기 시 현재 선택(조회 중인) 장부에만 반영하지 않고 다른 장부에 넣지 마 → 항상 현재 장부 기준
- [ ] 엑셀 가져오기 시 미리보기 확인 없이 기존 데이터를 덮어쓰지 마
- [ ] PDF 합계에 `final_amount` 대신 `subtotal`을 사용하지 마
- [ ] PDF 합계표를 별도 페이지로 분리하지 마 → 1페이지 상단에 배치
- [ ] 한글 엑셀 내보내기 시 UTF-8 BOM 누락하지 마
- [ ] 영수증 사진을 서버 로컬에 저장하지 마 → Supabase Storage 사용
- [ ] 유료 OCR API를 도입하지 마 → Tesseract.js 사용
- [ ] package.json 의존성 버전을 임의로 변경하지 마
- [ ] 초기 master 계정 없이 시스템을 배포하지 마 → 시드 데이터 필수
- [ ] Storage에 경로 규칙 없이 이미지를 저장하지 마 → `receipts/{department}/{user}/{receipt}_{ts}.jpg`
- [ ] 빈 상태 화면을 빈 화면으로 두지 마 → 안내 메시지 + 행동 유도 버튼
- [ ] 비동기 작업 중 로딩 표시 없이 두지 마 → 스켈레톤/스피너/프로그레스 바
- [ ] 버튼 중복 클릭을 허용하지 마 → 요청 중 버튼 비활성화

### 3D 효과

| 적용 위치 | 효과 | 구현 |
|----------|------|------|
| 카드 호버 | 살짝 떠오르는 느낌 | `hover:-translate-y-1 hover:shadow-md transition-all duration-200` |
| 대시보드 요약 카드 | 은은한 입체감 | `shadow-lg` + 그라데이션 배경 |
| 버튼 클릭 | 눌리는 느낌 | `active:scale-[0.97] transition-transform` |
| 영수증 이미지 카드 | 호버 시 떠오름 | `hover:-translate-y-0.5 hover:shadow-lg transition-all` |

> **원칙**: `translate-y`와 `shadow`로 은은한 깊이감만. `perspective`·`rotateX/Y` 같은 과도한 3D 변환은 사용하지 않음.

### 모션·애니메이션

| 인터랙션 | 모션 | 시간 |
|---------|------|------|
| 버튼 호버/클릭 | 색상 전환 + scale | 150ms |
| 카드 호버 | 떠오름 (translate-y + shadow) | 200ms |
| 페이지 전환 | 페이드 인 | 200~300ms |
| 모달 열기 | 스케일 업 (0.95 → 1) + 페이드 인 | 200ms |
| 모달 닫기 | 스케일 다운 (1 → 0.95) + 페이드 아웃 | 150ms |
| 토스트 등장 | 우측에서 슬라이드 인 | 300ms |
| 토스트 퇴장 | 페이드 아웃 | 200ms |
| 목록 아이템 등장 | 아래에서 페이드 업 (stagger) | 각 50ms 딜레이 |
| 숫자 변화 (잔액 등) | 카운트 업 애니메이션 | 400ms |
| 드롭다운 메뉴 | 스케일 다운 → 업 + 페이드 | 150ms |
| 상태 뱃지 변경 | 색상 트랜지션 | 200ms |

> **원칙**:
> - 모든 인터랙티브 요소에 `transition` 적용 (정적 느낌 방지)
> - 200~300ms 범위 유지 (너무 느리면 답답, 너무 빠르면 인지 불가)
> - `prefers-reduced-motion` 미디어 쿼리 존중 (접근성)
> - 페이지 로드 시 목록 stagger 애니메이션으로 동적인 첫인상

### 하지 말 것

- 다크 모드 구현하지 않음 (라이트 모드 전용)
- 네온 색상, 형광 컬러 금지
- `perspective`·`rotateX/Y` 등 과도한 3D 변환 금지
- 500ms 이상의 느린 애니메이션 금지
- 무한 반복 애니메이션 금지 (로딩 스피너 제외)
- 아이콘 과다 사용 금지 — 의미가 명확한 곳에만 (lucide-react 사용)

## UX/UI 세부 규칙

### 영수증 제출 (교사 - 스마트폰)
- **1건씩 제출**: 사진 1장 업로드 → OCR 결과 확인·수정 → 제출 (다중 제출 없음)
- **이미지 자동 압축**: 업로드 시 장변 1200px, 품질 80%로 리사이즈 (원본 5MB → 200~300KB)
- **지원 형식**: JPG, PNG, HEIC(아이폰) → 업로드 시 JPG로 변환
- **OCR 처리 중**: 로딩 스피너 표시, "인식 중..." 텍스트
- **OCR 실패 시**: 빈 칸으로 두고 수동 입력 유도 (에러 아님, "직접 입력해주세요" 안내)
- **카메라/갤러리**: 모바일에서 input[type=file]의 capture 속성으로 카메라 직접 촬영 또는 갤러리 선택

### 영수증 이미지 관리
- **개별 다운로드**: 이미지 확대 뷰에서 다운로드 버튼 → 원본(압축 후) 이미지 저장
- **이미지 교체**: 제출자 본인 또는 회계 교사 이상이 새 사진으로 대체 가능. 교체 시 기존 이미지는 Storage에서 삭제
- **이미지 삭제**: 제출자 본인 또는 회계 교사 이상이 삭제 가능. 삭제 후 image_url = null, 장부항목에는 영향 없음
- **승인 후 수정 제한**: 승인 완료(approved) 상태의 영수증 이미지는 회계 교사 이상만 교체·삭제 가능 (제출자 본인 불가)

### 장부 편집 (회계 교사 - PC)

**모달 다중 입력 방식** — 모달 안에 테이블 형태로 여러 건을 한 번에 입력

```
[모달 내부 레이아웃 — 입력 3칸 + 자동 1칸]
날짜      항목명          금액       카테고리 (자동)
3/25     이자            20,000    이자 (수입) ✔
3/25     셀 간식비        48,000    간식비 (지출) ✔
3/26     교육위원회 지원금  50,000    교육위원회 (수입) ✔
3/26     드럼 스틱        15,000    ▾ (수동 선택 필요)
3/27     ▍
                                    [전체 저장] [취소]
```

- **입력 순서**: 날짜 → 항목명 → 금액 → 카테고리 (3칸 수기 입력 + 1칸 자동)
- **카테고리 자동 추천**: 항목명 입력 완료 시 키워드 매칭으로 자동 선택
  - "간식" → 간식비 (지출), "이자" → 이자 (수입), "수련회" → 교육행사비 (지출) 등
  - 매칭 실패 시 빈 칸 → 사용자가 드롭다운에서 수동 선택
  - **자동 선택된 카테고리는 언제든 클릭하여 수정 가능** (자동 추천은 참고용, 최종 결정은 회계 교사)
  - 키워드 목록은 master가 카테고리 관리에서 편집 가능
- **자동 추천 ON/OFF 토글**: 장부 편집 화면 상단에 "카테고리 자동 추천" 토글 스위치
  - ON (기본값): 항목명 키워드로 카테고리 자동 선택
  - OFF: 카테고리 항상 빈 칸 → 매번 수동 선택
  - 설정은 사용자별로 저장 (브라우저 로컬스토리지)
- **수입/지출 자동 판별**: 카테고리의 type(income/expense)으로 자동 분류 (별도 버튼 불필요)
- 카테고리 드롭다운에 "(수입)" "(지출)" 라벨 표시
- **카테고리 드롭다운 UX**: Tab 진입 시 자동 열림, 화살표(↑↓) 선택, 타이핑 필터링

- **Enter**: 현재 행 확정 → 바로 아래에 새 입력 행 추가 (모달 유지)
- **전체 저장**: 입력한 여러 건을 한 번에 저장 → 장부 테이블에 날짜순 자동 정렬

- 금액 필드는 숫자만 입력, 자동 콤마 표시

## 초기 설정 (부트스트랩)

> 시스템 최초 배포 시 master 계정이 없으면 아무도 승인할 수 없는 문제 해결.

**최초 master 계정 생성 방법:**
1. Supabase 대시보드에서 직접 Auth 사용자 생성
2. DB의 `users` 테이블에 해당 UID로 레코드 삽입: `role = 'master'`, `status = 'active'`, `department_id = '고등부'`
3. 이후 이 master 계정으로 로그인하여 다른 사용자 승인

**초기 시드 데이터:**
- 기본 카테고리 9종 (02_DATA_MODEL.md 참고)
- 기본 직분 5종 (교사, 부장교사, 총무교사, 회계교사, 담당교역자)
- 전체회기 장부 1개 (type: "main", name: "전체회기", is_active: true)

> SQL 시드 스크립트를 `supabase/seed.sql`에 작성하여 초기 데이터 자동 삽입.

## 세션 관리

- **세션 유지**: Supabase Auth 기본 (JWT, 1시간 만료 + 자동 갱신)
- **자동 로그아웃**: 브라우저 탭을 닫으면 세션 유지 (remember me 기본 활성)
- **수동 로그아웃**: 프로필 메뉴에서 로그아웃 버튼
- **세션 만료 시**: "세션이 만료되었습니다. 다시 로그인해주세요." 토스트 → 로그인 페이지로 이동

## 편의 기능 (Phase 1에 포함)

### 미저장 변경 감지
- 장부 편집 모달에서 입력 중 **Esc / 닫기 / 브라우저 뒤로가기** 시: "저장하지 않은 입력이 있습니다. 정말 나가시겠습니까?" 확인 다이얼로그
- 영수증 제출 폼에서 사진 업로드 후 뒤로가기 시에도 동일

### 영수증 제출 임시 저장
- 사진 업로드 + OCR 완료 후 제출 전 상태를 **브라우저 localStorage에 자동 저장**
- 실수로 페이지를 닫아도 재접속 시 "작성 중인 영수증이 있습니다. 이어서 작성하시겠습니까?" 안내
- 제출 완료 시 임시 저장 데이터 삭제

### 일괄 승인
- 미승인 목록에서 **체크박스로 여러 건 선택 → [선택 승인] 버튼**
- 개별 검토가 필요 없는 소액 건이나 확인 완료된 건을 빠르게 처리
- 일괄 승인 시에도 확인 다이얼로그 표시 ("N건을 승인하시겠습니까?")

### 날짜 입력 편의
- 날짜 필드에 달력 피커 + **"오늘" / "어제" 바로가기 버튼**
- 장부 편집 모달에서: 이전 행의 날짜를 다음 행에 자동 복사 (연속 입력 시 편리)

### PDF 파일명 자동 생성
- 다운로드 시 파일명: `{부서명}_{결산기제목}.pdf`
- 예: `고등부_2026년전반기결산.pdf`

### 장부항목 ↔ 영수증 바로가기
- 장부 테이블에서 영수증 연결 항목(source: "receipt")에 📎 아이콘 표시
- 클릭 시 해당 영수증 상세 화면으로 이동
- 영수증 상세에서도 "장부에서 보기" 링크로 해당 장부항목으로 이동

### 영수증 중복 경고
- 같은 날짜 + 같은 금액의 영수증이 이미 존재할 경우: "동일 날짜·금액의 영수증이 이미 있습니다. 그래도 제출하시겠습니까?" 경고
- 제출을 막지는 않음 (실제로 같은 날 같은 금액 지출 가능)

## 접근성 (Accessibility) 기본 규칙

> 법적 의무는 없으나, 기본적인 키보드·스크린 리더 대응으로 사용성을 높인다.

### 키보드 탐색
- 모든 버튼·링크·입력 필드에 `Tab` 키로 이동 가능
- `Enter` / `Space`로 버튼 동작
- `Esc`로 모달·드롭다운 닫기
- 포커스 표시: `focus-visible:ring-2 focus-visible:ring-indigo-500` (마우스 클릭 시에는 링 미표시)

### ARIA 라벨
- 아이콘 전용 버튼: `aria-label` 필수 (예: 삭제 버튼 → `aria-label="삭제"`)
- 상태 뱃지: `role="status"` + 텍스트 포함 (색상만으로 상태 전달 금지)
- 모달: `role="dialog"` + `aria-labelledby`로 제목 연결
- 토스트 알림: `role="alert"` + `aria-live="polite"`

### 색상 대비
- 본문 텍스트: 배경 대비 4.5:1 이상 (WCAG AA 기준)
- 현재 컬러 시스템(Gray-800 on White, Indigo-600 on White)은 기준 충족
- 상태를 색상으로만 구분하지 않음 — 텍스트 라벨 또는 아이콘 병기

### 모션 접근성
- `prefers-reduced-motion: reduce` 감지 시 모든 애니메이션 비활성화 (이미 디자인 섹션에 명시)

## 무료 한도 관리

> Supabase 무료 플랜 한도 초과를 사전에 방지한다.

### 한도 현황

| 자원 | 무료 한도 | 위험 수준 |
|------|----------|----------|
| DB | 500MB | 영수증 수백~수천 건 수준이면 충분, 당분간 안전 |
| Storage | 1GB | 사진 200장(압축 후 5MB 평균) = 1GB → **가장 먼저 도달 가능** |
| Auth | 50,000 MAU | 교회 내부 사용, 초과 가능성 없음 |
| 대역폭 (Vercel) | 월 100GB | 소규모 사용, 초과 가능성 낮음 |

### 대응 방침

- **이미지 압축 강화 (Phase 1 기본 적용)**: 업로드 시 장변 1200px, 품질 80% → 원본 5MB가 200~300KB로 축소. 이 설정이면 Storage 1GB에 약 3,000~5,000장 저장 가능
- **master 대시보드에 Storage 사용량 표시 (Phase 1)**: Supabase API로 현재 사용량 조회 → "저장 공간: 420MB / 1GB (42%)" 형태로 표시
- **80% 도달 시 경고**: master 대시보드에 노란색 경고 배너 표시 ("저장 공간이 부족해지고 있습니다")
- **한도 초과 시**: 업로드 차단 + "저장 공간이 부족합니다. 관리자에게 문의해주세요" 에러 메시지 (이미 에러 처리 섹션에 정의됨)
- **장기 대응 (운영 후 판단)**: Supabase Pro 플랜 전환(월 $25, 8GB Storage) 또는 오래된 이미지 아카이빙
