# 고등부 영수증 관리 시스템 -- PRD

> 생성일: 2026-03-19

---

## 1. 제품 개요

### 한 줄 요약
교사들이 스마트폰으로 영수증을 제출하면 OCR이 자동 인식하고, 회계 교사가 승인하면 장부에 자동 반영되며, 담당 교역자가 전체 시스템과 권한을 관리하는 교회 고등부 회계 시스템.

### 해결하는 문제
- 교사들이 영수증을 따로따로 회계 교사에게 전달하는 과정이 번거로움
- 회계 교사가 받은 영수증을 엑셀에 수기 입력해야 함
- 반기 결산 시 영수증 + 합계를 A4 최소 장수로 출력하기 어려움
- 담당 교역자가 지출 현황을 한눈에 파악하기 어려움

### 핵심 가치
- **스마트폰 제출**: 교사가 현장에서 영수증 사진을 찍어 바로 제출, OCR 자동 인식
- **승인 → 장부 자동 반영**: 회계 교사가 승인하면 별도 입력 없이 장부에 즉시 추가
- **종이 절약 PDF**: 합계표 + 영수증 이미지를 같은 페이지에 배치, 4항목이면 A4 1장
- **역할 기반 권한**: 담당 교역자가 시스템 접근 권한을 부여·관리

## 2. 사용자

### 역할 구조

```
마스터 (master) — 개발자/시스템 슈퍼유저 (hyun)
  ├── 교육목사 (sub_master) — 회원관리 전담 운영자 (사용자 승인·역할 부여만)
  ├── 교육위원장 (auditor)  — 전체 부서 열람 전용 (재정 감사/실태조사)
  ├── 회계 교사 (accountant) — 자기 부서 장부·영수증 CRUD
  └── 교사 (teacher)         — 자기 부서 열람 + 영수증 제출
      (담당교역자, 부장교사, 총무교사, 일반교사 등은 모두 직분일 뿐, 시스템 역할은 teacher)
```

> **직분(position)과 역할(role)은 다른 개념:**
> - 직분: 교회 내 직책 (담당교역자, 부장교사, 회계교사 등) — 표시용
> - 역할: 시스템 권한 (master / sub_master / accountant / auditor / teacher) — 기능 접근 제어용
> - 예: 직분이 "담당교역자"라도 시스템 역할이 teacher이면 열람+제출만 가능
>
> **master는 교회 조직 외부의 개발자 계정.** 실제 교회 운영은 accountant가 담당.

> **accountant**: 담당 회계교사. 자기 부서 재정 실무 전권.
> **teacher**: 일반교사. 영수증 제출 + 본인 조회.

### 사용자 시나리오

**시나리오 1 — 계정 신청 & 승인**
1. 교사가 시스템 URL을 받고 접속
2. 계정 신청: 아이디(이메일) + 비밀번호 + 실명 + 직분 입력
3. "승인 대기중" 화면 표시 — 시스템 접근 불가
4. 마스터가 신청 대기 목록에서 확인 → 승인 + 시스템 역할 부여 (teacher/accountant)
5. 승인된 교사가 로그인하면 역할에 맞는 기능 사용 가능

**시나리오 2 — 교사가 영수증 제출 (스마트폰, 1건씩)**
1. 승인된 교사가 로그인
2. 영수증 사진 1장 촬영 또는 갤러리에서 선택 (자동 압축)
3. OCR이 날짜·금액 자동 입력 (인식 실패 시 빈 칸 → 수동 입력)
4. 내용·분류 확인·수정 후 제출
5. 이후 자신의 제출 내역에서 상태 확인 (대기중/승인됨/반려됨)

**시나리오 3 — 회계 교사가 검토·승인 (PC)**
1. 미승인 영수증 목록에서 제출 건 확인
2. OCR 인식 결과 검토, 필요 시 금액·내용 수정
3. 승인 → 장부에 자동 추가됨
4. 이체는 카카오뱅크 등 외부에서 별도 처리

**시나리오 4 — 반기 결산 PDF 출력 (감사 제출용)**
1. 결산 메뉴 → 기간 선택 (사용자가 직접 설정한 결산 기간)
2. PDF 미리보기: 상단 합계표 + 하단 영수증 이미지 (최소 페이지)
3. PDF 다운로드 → 인쇄 후 영수증 철

**시나리오 5 — 엑셀 연동**
1. 기존 엑셀 장부 가져오기 → 컬럼 매핑 확인
2. 이후 작업은 이 시스템에서 진행
3. 필요 시 현재 데이터 엑셀로 내보내기

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 — 제출 → 승인 → 장부

```
[교사 - 스마트폰]              [회계 교사 - PC]
  로그인                          로그인
  → 사진 업로드                   미승인 목록 확인
  → OCR 자동 입력                 → 검토·수정
  → 확인 후 제출                  → 승인 클릭
                                  → 장부항목 자동 생성 ✅
  내 제출 내역에서
  상태 확인 가능
```

### 핵심 흐름 — 장부 관리

```
장부 선택 (전체회기 / 수련회 / 전도축제 등)
  → 장부항목 조회 (날짜·분류 필터 + 행별 누적 잔액)
  → 행 추가·수정·삭제 (엑셀처럼 직접 편집)
  → 특수 장부 새로 생성·종료
```

### 핵심 흐름 — 감사용 PDF 출력

```
결산 메뉴 → 결산기 선택 → PDF 미리보기 → 다운로드 → 인쇄 → 철
```

PDF 구성 (종이 최소화):
```
[1페이지 — 항목 4개 이하일 때 A4 1장 완료]
┌──────────────────────────────────────┐
│ 2026년 전반기 결산 | 고등부           │
│ 식비 ₩125,000 / 교통비 ₩45,000     │ ← 합계표 (상단)
│ 총계 ₩348,000                       │
├──────────────────┬───────────────────┤
│ 3월 셀 간식비     │ 수련회 교통비     │ ← 항목명 (소제목)
│  [영수증 이미지]  │  [영수증 이미지]  │ ← 이미지 (크게)
│ ₩48,000          │ ₩12,000          │ ← 최종금액
├──────────────────┼───────────────────┤
│ 교재 구입         │ 사무용품          │
│  [영수증 이미지]  │  [영수증 이미지]  │
│ ₩33,000          │ ₩7,000           │
└──────────────────┴───────────────────┘
```

## 5. 성공 기준

- [ ] 교사가 영수증 제출에 2분 이내 완료 (사진 촬영 포함)
- [ ] 회계 교사가 승인 즉시 장부에 반영됨 (수동 입력 없음)
- [ ] 4항목 이하 결산은 A4 1장으로 출력됨
- [ ] 교사가 자신의 제출 내역 상태를 확인할 수 있음
- [ ] 담당 교역자가 원하는 사용자에게 권한을 부여/삭제할 수 있음

## 6. 안 만드는 것 (Out of Scope)

- 이체 처리 기능 — 카카오뱅크 등 외부 서비스 이용, 이 시스템은 승인까지만
- 푸시 알림 — Phase 2 이후 검토
- 네이티브 모바일 앱 — 반응형 웹으로 충분
- 다른 부서 연동 — Phase 3에서 확장

## 7. 확정 사항

- ✅ 엑셀 장부 컬럼: 항목, 수입액, 지출액, 잔액, 카테고리 — LedgerEntry 매핑 (02_DATA_MODEL.md 참고)
- ✅ 결산기 기간: master가 직접 설정 (고정 아님)
- ✅ 결산·PDF 기준: LedgerEntry 기준 집계, 지출만 PDF 출력
- ✅ 역할 체계: master(최고권한) / accountant(자기 부서 장부 CRUD) / teacher(열람+제출) / auditor(Phase 3, 전체 열람)
- ✅ 직분과 역할은 별개: 직분(담당교역자 등)은 표시용, 역할은 권한 제어용
- ✅ 계정 신청 방식: 아이디(이메일) + 비밀번호 + 실명 + 직분 → master 승인 + 시스템 역할 부여 (부서 선택 UI는 Phase 2)
- ✅ 장부 구조: 전체회기(main) + 특수 목적(special) 장부, 독립 관리
- ✅ 영수증↔장부항목 독립: 영수증 수정/삭제 시 장부항목 영향 없음
- ✅ 직분 목록: 기본 제공 + master 관리 가능
- ✅ 교사 장부 열람: 읽기 전용 접근 가능
- ✅ `department_id`: Phase 1에서 "고등부" 자동 배정, Phase 3에서 테이블 전환
